# 0.0.1 — 最小静态 OK 状态页

## 范围

- 单个静态 HTML 页面，居中显示大字「OK」
- 纯静态 HTML + CSS，无 JavaScript 框架
- 部署到 Aliyun `/projects/console-e2e-probe/` basePath
- HTML 文档 `Cache-Control: no-cache`
- 静态资源带 `?v=0.0.1` 版本令牌

## 关联输入

- `e000001` owner_input: "做一个最小静态 hello 状态页：单页显示大字 OK 即可"
- `e000002` requirement_defined
- `e000003` owner_confirmation

## 验收标准

1. 公网 URL 浏览器访问成功，HTTP 200
2. 页面居中显示大字 OK，字体大小 ≥ 48px
3. CSS 样式正常加载，无 404
4. 浏览器控制台无错误
5. Kimi 视觉审查通过

## 技术方案概要

纯静态 HTML + CSS，单文件。本地用 Python http.server 开发，生产用 Nginx 直接 serve 静态文件。

## 实施任务拆解

### T1: 创建 `src/index.html`

- 单文件纯静态 HTML5
- `<meta http-equiv="Cache-Control" content="no-cache">` 满足 no-cache 要求
- 内联 `<style>` 实现深色背景 + 居中布局
- Flexbox 居中，`min-height: 100vh`
- `<h1>OK</h1>`，字号 72px（≥ 48px），白色文字
- 深色背景 `#0d1117`（GitHub dark 风格），简洁美观
- 页面标题 `Console E2E Probe — Status`
- 无外部 CSS/JS 依赖，无需 `?v=` 令牌（无外部资源引用）

### T2: 自测验证

- `cd src && python3 -m http.server 8080 &`
- curl 验证: HTTP 200、内容包含 OK、Cache-Control meta 标签存在
- 记录自测输出到 `evidence/claude/`

### T3: 提交

- Git commit，遵循小步提交
